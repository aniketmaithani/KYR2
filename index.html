The following document lays out the API structure for the KYR Application



API ENDPOINTS

All the Request/Response data is in application/json format unless explicitly specified

/search?q=:PARAMS:

This endpoints allows user's to search for their respective MP's .


CALL TYPE : POST

Request :

{ "candidate_name":"abcd" }



Response :

{ "name": "abasadlkasdjalsdasld" "nature_of_membership": "abasadlkasdjalsdasld" "start_term": "abasadlkasdjalsdasld" "end_term": "abasadlkasdjalsdasld" "state": "abasadlkasdjalsdasld" "political_party": "abasadlkasdjalsdasld" "gender": "male" "educational_qualification": "abasadlkasdjalsdasld" "age": "45" "debates_participated": "27" "private_member_bills": "45" "questions_asked": "45" "attendance": "45" "other_notes": "abasadlkasdjalsdasld" }



/complaint

This endpoints allows user's to send complain to the respective MP on social media or on our account on Twitter.


CALL TYPE : POST

Request :

{ "name_of_the_person_compalining":"abcd" "phone_number_of_the_person":"+910132646544" "photo_of_the_area": "https://example.com/abc.jpg" (Refers to the photo of the area about which user wants to complain) "description_of_the_problem": "lorem ipsum abcd lkasjdlaksjd lksajdl;skjasd lkfjsdff " (word limit upto 100 words) "geolocation": "14.04N, 35.04E" "name_of_the_mp_for_problem_redressal": "abcded" }



Response : 200 OK

{ "status":"submitted" "concerned_mp":"abcd" "complain_id": "ARN101012123" }


After a while the user's complain is posted along with the given complain_id on twitter with appropriate hashtags by our admin team. The user can check it on Twitter using complain_id as a hashtag
